# Mebug Visual Guidelines

Mebug 的视觉基调来自现有品牌插画：暖奶油底、桃粉花朵、鼠尾草绿叶、木质浅棕、深可可文字。整体目标是柔和、生活化、轻阅读，不做高对比商业站，也不做单一棕色长文。

## 色彩基调

| 用途 | Token | 色值 | 使用场景 |
| --- | --- | --- | --- |
| 页面背景 | `--bg` | `#fff8ef` | 页面底色 |
| 卡片底色 | `--paper` | `#fffdf8` | 文章卡片、图片容器 |
| 主文字 | `--text-heading` | `#513832` | 站名、文章标题、重要标题 |
| 正文 | `--text-body` | `#604a43` | 文章正文段落 |
| 柔和正文 | `--text-body-soft` | `#786156` | 摘要、副标题、说明文字 |
| 辅助文字 | `--text-muted` | `#907b70` | 时间、计数、页脚、导航弱文字 |
| 次级说明 | `--text-subtle` | `#ad9587` | 极弱辅助信息，慎用 |
| 绿色强调 | `--text-accent` | `#69753e` | 标签、英文 eyebrow、链接默认色 |
| 暖色强调 | `--text-warm` | `#a85f52` | 日期、小节标题、悬停强调 |
| 链接文字 | `--text-link` | `#69753e` | 正文链接、上一篇/下一篇 |
| 行内代码 | `--text-code` | `#7c4f37` | 行内代码文字 |
| 反白文字 | `--text-inverse` | `#fff8ef` | 深色代码块文字 |

## 文字层级

- `H1 / 站名 / 文章标题`：使用 `--text-heading`，保持强识别，但不要用纯黑。
- `H2 / H3`：使用 `--text-heading`，配合顶部细分隔线制造章节节奏。
- `H4 / 自动小节标题`：使用 `--text-warm`，背景用淡桃粉，适合 `**1. 关于...**` 这类段落。
- `正文段落`：使用 `--text-body`，避免长文全部深色造成疲劳。
- `摘要和说明`：使用 `--text-body-soft` 或 `--text-muted`。
- `链接`：默认鼠尾草绿 `--text-link`，hover 切换暖桃棕 `--text-warm`。
- `标签`：背景 `--accent-soft`，文字 `--text-accent`。

## 文章排版规则

- 文章详情页版心为 `1080px`，移动端自动收窄。
- 正文使用非衬线字体：`Inter / system-ui / Noto Sans SC`。
- 正文行高保持 `1.85`，适合中文长文阅读。
- 支持 `#` 到 `#####` 五级标题，一级到三级用章节分隔线，四级作为浅桃色小节标签，五级作为鼠尾草绿小标题。
- 支持无序列表、有序列表、缩进子项，列表 marker 使用暖桃棕。
- 支持 `---` 分割线，显示为柔和水平线。
- 连续 Markdown 图片会自动组成 `.photo-grid`，桌面多列，移动端单列。
- 单独的加粗段落 `**关于工作**` 会自动转成 `h4` 小节标题。
- 首页文章列表按年份分组，每个年份下保持三列卡片布局。

## 品牌资源

- 头部导航标识：`public/brand/logo-mark.png`
- 备用横向品牌图：`public/brand/logo.png`、`public/brand/logo-nav.png`
- favicon：`public/favicon.png`

不要直接引用 `content/drafts/` 中的图片作为线上资源；需要发布的图片应放在 `public/` 下。

## 可维护性

- 所有视觉 token 维护在 `src/styles/site.css` 的 `:root` 中。
- 不要在组件区域随手新增硬编码文字色，优先复用 `--text-*` tokens。
- 修改源样式后必须运行 `npm run publish`，因为 `docs/` 是生成输出。
