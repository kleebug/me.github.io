午后的阳光洒在木质圆桌上，我刚刚修剪完一株月季。看着地上散落的枝叶，忽然觉得这和写文章很像——*只有修剪掉多余的词句，文字才能透进光来*。昨天给新买的 **iPhone 15** 设置完壁纸，心情也跟着明亮起来。（注意这里的中英混排规范：**iPhone 15** 前后留有半角空格）。

> 一棵植物的呼吸，隐藏在它愿意为光留出的缝隙里。
    一篇文字的温度，停留在排版为视线留出的留白处。


## 🌿 一、细节决定了整体氛围

好的排版，就像照料花园，全在细微处。比如行内的 **高亮强调**，用来标记核心观点；比如 `行内代码`，用来优雅地标注极简的专业名词；又比如偶尔使用 ~~删除线~~，在文末调侃自己过去的错漏，反而多了一丝真实感。

这正是规则要赋予文章的结构感。在长篇大论中，我通常会安排这样几个层次的表达：

- **呼吸感** —— 善用 `blockquote` 和 `info-box` 让视线停顿。
- **秩序感** —— 使用带有粉色底色的 `table` 表头来整理数据。
- **画面感** —— 给图片加上 16px 圆角及清晰优雅的 `figcaption` 图注。

### （一）关于颜色规范的具体色谱

为了统一视觉，我将整个 Kleebug 的配色方案做成了规范表，便于后续对接时参考：

| 元素类型 | 色值代码 | 使用场景 |
| :--- | :--- | :--- |
| 深陶土棕 | `#6A4C46` | 所有标题、重点图标 |
| 浅粉 / 深粉 | `#F3C2C6` / `#EBA0A5` | H2 下划线、引用边、浅底说明框 |
| 鼠尾草绿 | `#A8BFA2` | 列表星号、元数据边线 |
| 暖奶白 / 浅橙 | `#FEF8F0` / `#FFF0E8` | 全局背景、行内高亮背景 |

### （二）代码块与说明框的优雅呈现

如果我们想在文章里分享一段 CSS 代码片段，它的呈现方式如下（背景与引用块对齐）：

/* Kleebug CSS 核心片段 */
.highlight {
    background: #FFF0E8;
    padding: 2px 8px;
    border-radius: 6px;
}

‘’‘这是代码块设计，看看信不信’‘’

[!NOTE]
💡 细节补充：关于文章的结尾结构
按照规范要求，文章主体结束后，要用渐变粉色 hr 分割线做一个视觉断开，然后接上“文末小记”。如果遇到特别补充的内容，就用这种 info-box 呈现，它的“微阴影”和粉边浅底，能很好地与周围内容区分开来。
